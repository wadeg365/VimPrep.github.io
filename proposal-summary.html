BigQuery Lakehouse Migration Proposal

A comprehensive summary of Datatonic's proposal to migrate Vimeo's Snowflake platform to Google Cloud

Proposal Overview

Datatonic, a leading Google Cloud partner for data + AI, has proposed a migration of Vimeo's current data platform, business intelligence (BI), and artificial intelligence (AI) solutions from Snowflake Enterprise Data Warehouse (EDW) to a modern data architecture in Google Cloud. The proposal aims to solve Vimeo's biggest challenges and unlock true business value through a comprehensive migration approach.

The migration from AWS to Google Cloud will occur in two phases:

  • An initial 3-week design phase to confirm timescales
  • A build phase of approximately 6 months, including System Integration Testing (SIT) and User Acceptance Testing (UAT)

Migration Timeline & Phases

Phase 1: Solution Design

Review and analyze current state in Snowflake, identify priority use cases for accelerated end-to-end platform build, and produce logical data flow designs tailored for BigQuery.

Duration: ~3 weeks

Phase 2: MVP Implementation

Productionize MVP use cases and corresponding data models, integrate with source systems, and develop automation, security, and governance frameworks.

Duration: ~12 weeks

Phase 3: Scale Out

Scale the new solution to migrate remaining use cases, data models, and ETL pipelines to Google Cloud. Begin Snowflake decommissioning plan and co-development with Vimeo team.

Duration: ~12 weeks

Post Go-Live

Complete knowledge transfer, provide ongoing support, and ensure full platform adoption across the organization.

Duration: ~8 weeks

Key Technology Components

The proposed target architecture leverages key Google Cloud technologies to create a modern, scalable data platform:

Data Lakehouse

Migrating database tables, views, and routines to BigQuery, Google's world-class solution for data warehousing and analytics.

"Big Lake" For Raw Data

Utilizing a unified approach to data lake and data warehouse to enable building a true, frictionless data lakehouse in BigQuery.

Batch Data Transformation

Employing Dataform, Google's fully managed & serverless SQL transformation tool for SQL transformation and data model schemas.

ELT Workflow Orchestration

Managing ELT/ETL workflows orchestrated by Cloud Workflows for efficient and reliable data processing.

Architecture Overview

Achieving a successful migration to BigQuery requires building key pillars of a modern data architecture on a solid foundation:

The new target data architecture in BigQuery is designed to be:

  • Scalable to handle Vimeo's growing data volumes
  • Cost-effective with optimized resource utilization
  • Highly performant for analytics workloads
  • Future-proof with modern data management capabilities

The migration will include translating business and transformation logic from legacy SQL to BigQuery SQL dialect, ensuring compatibility and optimized performance.

Delivery Approach

Datatonic's approach follows agile principles with a focus on rapid development, flexibility, and reduced risk:

  • Agile ways of working: 2-week sprints with backlog refinement and sprint planning, increased flexibility, parallel working, and continuous improvement.
  • Platform, security and governance shifted left: Prioritizing platform services upfront, focusing on security and governance infrastructure, and enabling business value earlier through iterative deployments.
  • Prioritization based on Impact: Implementing use cases as iterations rather than big bang deployments, enabling earlier testing and UAT, and reducing risk over time/budget.

The recommended approach follows a discovery phase and rapid MVP implementation to maximize value:

  1. Design: Discovery, Use Case Prioritization, Design, Exploratory Data Analysis
  2. Ingestion & Transformation: Prioritized Data Ingestion
  3. Platform MVP: Core infrastructure and services
  4. Product Increment: Iterative improvements with each Use Case
  5. Training & Enablement: Comprehensive knowledge transfer

Post-Migration Support

Datatonic offers comprehensive post-migration support to ensure success:

  • Org-Wide Training Programs: Supporting learners at every level (Business Leaders, Technical Personas, Business Users) with tailored approaches.
  • Managed Services: Engagement framework defining roles for maintenance, SLAs, continuous improvement, and knowledge sharing.
  • Quality Guarantee: Experienced engineers trained on best practices, security and compliance, with no offshoring.

The Foundry Model for building data capability includes:

  1. Datatonic initially owning end-to-end delivery
  2. Gradually co-developing with Vimeo's team
  3. Eventually rolling off, leaving a fully functioning data capability
